# 基于 IMX6ULL 的会员识别智能零售柜终端系统技术路线

> 项目定位：面向嵌入式 Linux 应用层岗位的综合型项目。  
> 推荐项目名：**基于 IMX6ULL 的会员识别智能零售柜终端系统** / **Smart Member Retail Cabinet Based on Embedded Linux**。

---

## 1. 项目背景与定位

本项目不再做传统意义上的“用户直接选择商品并出货”的无人售货机，而是调整为更真实、更好演示的**会员制智能零售柜**：

```text
会员通过摄像头登录 / 注册
        ↓
识别成功后柜门打开
        ↓
用户在终端界面选购商品
        ↓
点击创建订单
        ↓
系统从会员卡余额中扣款
        ↓
扣减商品库存
        ↓
订单完成
        ↓
柜门关闭
        ↓
订单、会员、库存、温度、门状态上报后台
```

这个项目的核心不是复杂 AI，也不是完整商业支付，而是通过“会员识别 + 智能柜门 + 商品订单 + 本地缓存 + 物联网通信 + OTA”串起嵌入式 Linux 应用层能力。

项目重点体现：

- Linux 多线程
- V4L2 摄像头采集
- 会员注册与会员识别流程
- 商品管理与库存管理
- 会员余额支付
- 舵机 / PWM 模拟柜门开关
- RS485 / Modbus RTU 温度传感器
- SQLite 本地持久化
- MQTT 设备通信
- TCP / HTTP 图片和数据传输
- LVGL 本地 UI
- Qt 后台管理
- OTA 远程升级
- 状态机、日志系统、异常处理、断网缓存与补传

---

## 2. 项目总体目标

实现一套运行在 IMX6ULL 开发板上的会员识别智能零售柜终端系统，具备以下能力：

1. 支持会员注册：通过 V4L2 摄像头抓拍会员头像 / 人脸图像，并保存会员信息。
2. 支持会员识别登录：通过摄像头抓拍并模拟识别会员身份。
3. 会员识别成功后，控制舵机模拟柜门打开。
4. 会员登录后才能进入商品选购页面。
5. 用户选择商品后点击创建订单。
6. 创建订单时从会员卡余额中扣款。
7. 支付成功后扣减商品库存。
8. 订单完成后控制舵机模拟柜门关闭。
9. 通过 RS485 / Modbus RTU 温度传感器采集柜内温度。
10. 柜内温度通过 MQTT 周期上报，超温时上报告警。
11. 使用 SQLite 保存会员、商品、订单、余额流水、门状态记录、温度记录和日志。
12. 支持断网订单缓存和网络恢复后自动补传。
13. 使用 MQTT 完成设备心跳、会员登录事件、订单事件、柜门状态、温度数据、告警和远程控制。
14. 使用 TCP / HTTP 完成会员图片、识别抓拍图片、订单相关图片上传。
15. 使用 Qt 后台完成会员管理、商品管理、订单管理、库存管理、设备监控、柜门控制、温度监控和 OTA 升级。
16. 支持应用级 OTA 升级，包括版本检测、升级包下载、SHA256 校验、安装替换和失败回滚。

## 5. 核心业务流程

### 5.1 会员注册流程

```text
用户点击“会员注册”
        ↓
输入会员名称 / 手机号 / 初始余额
        ↓
V4L2 摄像头抓拍会员头像
        ↓
保存会员图片到 data/member_faces/
        ↓
SQLite 保存会员信息
        ↓
MQTT 上报会员注册事件
        ↓
Qt 后台显示新增会员
```

第一版可以不做真实人脸特征提取，只做：

```text
抓拍图片 + 保存会员信息 + 模拟注册成功
```

增强版可以做：

```text
V4L2 抓拍 → HTTP 上传后台 → 后台 OpenCV 检测人脸 → 返回注册成功
```

---

### 5.2 会员识别登录流程

```text
用户点击“会员登录”
        ↓
V4L2 摄像头抓拍当前人脸图片
        ↓
第一版：模拟识别为固定会员
增强版：后台 OpenCV / 本地轻量识别
        ↓
识别成功
        ↓
加载会员信息和余额
        ↓
会员状态变为 LOGIN_SUCCESS
        ↓
柜门状态变为 OPENING
        ↓
舵机动作，柜门打开
        ↓
进入商品选购页面
```

识别失败：

```text
抓拍失败 / 识别失败
        ↓
会员状态变为 LOGIN_FAILED
        ↓
柜门保持关闭
        ↓
UI 提示“会员识别失败，请重试”
        ↓
MQTT 上报识别失败事件
```

---

### 5.3 会员选购与余额支付流程

```text
会员登录成功
        ↓
柜门已打开
        ↓
进入商品选购页面
        ↓
用户选择商品
        ↓
点击“创建订单”
        ↓
系统检查商品库存
        ↓
系统检查会员余额
        ↓
余额充足
        ↓
从会员余额扣款
        ↓
扣减商品库存
        ↓
生成订单记录和余额流水
        ↓
订单状态变为 DONE
        ↓
柜门关闭
        ↓
MQTT 上报订单事件、余额变化、库存变化、柜门状态
        ↓
Qt 后台显示订单记录
```

余额不足：

```text
创建订单
        ↓
检查会员余额
        ↓
余额不足
        ↓
订单状态 FAILED
        ↓
UI 提示余额不足
        ↓
柜门关闭或保持打开等待用户退出，第一版建议关闭
        ↓
MQTT 上报订单失败事件
```

库存不足：

```text
选择商品
        ↓
检查库存
        ↓
库存为 0
        ↓
提示商品售罄
        ↓
不创建订单或创建失败订单
```

---

### 5.4 柜门控制流程

```text
会员识别成功
        ↓
door_open()
        ↓
舵机转到开门角度
        ↓
柜门状态 OPENED
        ↓
允许选购商品

订单完成 / 退出登录 / 超时
        ↓
door_close()
        ↓
舵机转到关门角度
        ↓
柜门状态 CLOSED
        ↓
会员退出登录
```

第一版用 mock：

```text
[DOOR] mock open cabinet door
[DOOR] mock close cabinet door
```

第二版接舵机：

```text
PWM 输出指定占空比 → 舵机转动 → 模拟柜门开关
```

---

### 5.5 柜内温度采集流程

```text
sensor_thread 周期运行
        ↓
通过 RS485 发送 Modbus RTU 读寄存器命令
        ↓
温度传感器返回温度寄存器数据
        ↓
解析柜内温度
        ↓
写入 app_context / SQLite
        ↓
MQTT 上报 cabinet_env
        ↓
LVGL 状态页显示柜内温度
        ↓
Qt 后台温度监控页面更新
```

超温告警：

```text
柜内温度 > 阈值
        ↓
生成温度告警
        ↓
蜂鸣器 / LED 提示
        ↓
MQTT 上报告警
        ↓
Qt 后台弹出告警记录
```

---

### 5.6 断网缓存与补传流程

```text
订单完成
        ↓
检测 MQTT / TCP 是否在线
        ↓
在线：立即上报订单、会员余额变化、库存变化
        ↓
离线：订单写入 SQLite，upload_status = pending
        ↓
网络恢复
        ↓
查询 pending 订单
        ↓
自动补传订单和相关事件
        ↓
补传成功后 upload_status = uploaded
```

---

### 5.7 OTA 升级流程

```text
Qt 后台点击推送 OTA
        ↓
后台发布 ota_notify MQTT 消息
        ↓
终端收到 OTA 通知
        ↓
检查当前是否空闲
        ↓
若会员正在购物，不立即升级
        ↓
空闲后下载 version.json
        ↓
比较版本
        ↓
下载升级包
        ↓
SHA256 校验
        ↓
备份旧程序
        ↓
替换新程序
        ↓
重启应用
        ↓
启动成功上报 OTA success
        ↓
启动失败自动回滚
```

## 6. 状态机设计

### 6.1 整机状态机 app_state

```c
typedef enum {
    APP_STATE_INIT = 0,
    APP_STATE_IDLE,
    APP_STATE_MEMBER_REGISTERING,
    APP_STATE_MEMBER_IDENTIFYING,
    APP_STATE_MEMBER_LOGGED_IN,
    APP_STATE_DOOR_OPENING,
    APP_STATE_SHOPPING,
    APP_STATE_ORDER_CREATING,
    APP_STATE_PAYING,
    APP_STATE_DOOR_CLOSING,
    APP_STATE_SUCCESS,
    APP_STATE_OFFLINE,
    APP_STATE_ERROR,
    APP_STATE_OTA_UPDATING
} app_state_t;
```

状态说明：

| 状态 | 含义 |
|---|---|
| INIT | 系统初始化 |
| IDLE | 空闲，等待会员登录 |
| MEMBER_REGISTERING | 会员注册中 |
| MEMBER_IDENTIFYING | 会员识别中 |
| MEMBER_LOGGED_IN | 会员已登录 |
| DOOR_OPENING | 柜门开启中 |
| SHOPPING | 商品选购中 |
| ORDER_CREATING | 创建订单中 |
| PAYING | 会员余额支付中 |
| DOOR_CLOSING | 柜门关闭中 |
| SUCCESS | 当前流程成功 |
| OFFLINE | 网络离线 |
| ERROR | 系统异常 |
| OTA_UPDATING | OTA 升级中 |

## 7. 线程模型设计

### 7.1 推荐线程架构

```text
main
 ├── ui_thread
 ├── member_thread
 ├── order_thread
 ├── door_thread
 ├── camera_thread
 ├── sensor_thread
 ├── mqtt_thread
 ├── tcp_upload_thread
 ├── ota_thread
 ├── storage_thread
 └── log_thread
```

### 7.2 各线程职责

| 线程 | 职责 |
|---|---|
| ui_thread | LVGL 刷新、处理会员登录、商品选购、订单状态显示 |
| member_thread | 会员注册、会员识别、会员登录状态管理 |
| order_thread | 订单创建、余额扣款、库存扣减、订单状态流转 |
| door_thread | 舵机 / PWM 控制柜门开关 |
| camera_thread | V4L2 摄像头抓拍，注册图和识别图保存 |
| sensor_thread | Modbus RTU 读取柜内温度 |
| mqtt_thread | MQTT 连接、订阅、发布、断线重连 |
| tcp_upload_thread | 图片上传、订单补传、大文件传输 |
| ota_thread | OTA 检查、下载、校验、安装、回滚 |
| storage_thread | SQLite 读写、缓存订单、查询 pending 数据 |
| log_thread | 异步日志落盘、日志轮转 |

## 13. 开发阶段规划

### 阶段 1：PC 端业务模拟

目标：先不依赖真实硬件，跑通核心业务。

内容：

1. app_state 整机状态。
2. product_manager 固定商品数据。
3. member_manager 固定会员数据和余额。
4. door_manager mock 开门 / 关门。
5. order_state 订单状态。
6. order_manager 创建订单、余额支付、扣库存。
7. main.c 命令行测试完整流程。

完成标准：

```text
会员模拟登录成功 → 柜门 mock 打开 → 选择商品 → 创建订单 → 会员余额扣款 → 商品库存扣减 → 柜门 mock 关闭。
```

---

### 阶段 2：SQLite 持久化

目标：让会员、商品、订单、余额流水落库。

内容：

1. SQLite 初始化。
2. member 表。
3. product 表。
4. orders 表。
5. balance_record 表。
6. door_record 表。
7. cabinet_env 表。
8. pending 订单查询和补传准备。

完成标准：

```text
每次购买后，订单、余额变化、库存变化可以在 SQLite 中查询。
```

---

### 阶段 3：LVGL 本地界面

目标：用屏幕完成会员登录和商品选购。

内容：

1. 首页。
2. 会员登录页。
3. 会员信息页。
4. 商品选购页。
5. 创建订单按钮。
6. 订单结果页。
7. 状态页显示柜门和温度。

完成标准：

```text
通过 LVGL 点击完成会员登录、商品选择、创建订单和支付流程。
```

---

### 阶段 4：V4L2 会员注册与识别

目标：让摄像头真正参与会员流程。

内容：

1. camera_if 抽象接口。
2. camera_mock。
3. camera_v4l2。
4. 会员注册抓拍。
5. 会员登录抓拍。
6. 抓拍图片保存到 member_faces。
7. 第一版模拟识别，增强版接后台 OpenCV。

完成标准：

```text
点击会员登录后，V4L2 抓拍图片，并完成模拟识别登录。
```

---

### 阶段 5：舵机模拟柜门

目标：用舵机实现开柜 / 关柜动作。

内容：

1. pwm_ctrl。
2. servo_ctrl。
3. door_manager。
4. door_state。
5. 会员登录成功开门。
6. 支付完成关门。
7. 超时未关门告警。

完成标准：

```text
会员识别成功后舵机转到开门角度，订单完成后舵机转到关门角度。
```

---

### 阶段 6：Modbus 温度传感器

目标：采集柜内温度。

内容：

1. 串口配置。
2. Modbus RTU 读寄存器。
3. 温度解析。
4. 周期采集。
5. 温度上报。
6. 超温告警。

完成标准：

```text
终端能周期读取柜内温度，并在 LVGL 和 Qt 后台显示。
```

---

### 阶段 7：MQTT 物联网通信

目标：让终端具备设备接入能力。

内容：

1. Mosquitto Broker 搭建。
2. 心跳上报。
3. 会员登录事件上报。
4. 订单事件上报。
5. 柜门状态上报。
6. 温度数据上报。
7. 告警上报。
8. 后台远程开门 / 关门 / 读温度。

完成标准：

```text
Qt 后台能看到设备在线、会员登录、订单、柜门状态和温度，并能远程控制柜门。
```

---

### 阶段 8：断网缓存与补传

目标：模拟真实设备网络异常。

内容：

1. 网络状态检测。
2. 离线订单缓存。
3. pending 订单查询。
4. 网络恢复后补传。
5. 补传成功更新状态。

完成标准：

```text
断网购买不会丢订单，网络恢复后订单自动上传。
```

---

### 阶段 9：OTA 远程升级

目标：实现应用级 OTA。

内容：

1. version.json。
2. MQTT OTA 通知。
3. HTTP 下载升级包。
4. SHA256 校验。
5. 备份旧程序。
6. 替换新程序。
7. systemd 重启。
8. 启动失败回滚。
9. OTA 状态上报。

完成标准：

```text
Qt 后台点击升级后，终端下载新版本程序并完成替换；如果升级失败，可以回滚旧版本。
```

## 16. CMake 构建建议

```text
build_pc/     # PC 本地模拟构建
```

```bash
mkdir -p build_pc
cd build_pc
cmake .. -DTARGET_PLATFORM=PC
make -j4
```

### 16.3 ARM 构建

```bash
mkdir -p build_arm
cd build_arm
cmake .. \
  -DTARGET_PLATFORM=ARM \
  -DCMAKE_TOOLCHAIN_FILE=../cmake/arm-linux-gnueabihf.cmake
```

### 17.3 项目描述

```text
基于 IMX6ULL 设计并实现会员识别智能零售柜终端系统，终端侧支持 V4L2 摄像头会员注册与识别、会员余额支付、商品选购、库存扣减、舵机模拟柜门开关、Modbus 温度采集、SQLite 本地缓存、MQTT 状态上报、TCP/HTTP 图片上传以及 OTA 远程升级；后台侧基于 Qt 实现会员管理、商品管理、订单管理、柜门控制、温度监控、MQTT 消息监控和 OTA 升级管理。
```

---

### 17.4 项目职责

```text
1. 设计终端侧多线程架构，实现 UI、会员识别、订单、柜门控制、温度采集、MQTT、OTA 等模块解耦。
2. 基于 V4L2 实现会员注册与登录抓拍功能，并设计会员登录状态与订单绑定流程。
3. 设计会员余额支付流程，实现订单创建、余额校验、余额扣减、库存扣减和订单状态流转。
4. 基于 PWM / 舵机模拟柜门开关，实现会员识别成功开门、支付完成关门的设备控制流程。
5. 基于 RS485 / Modbus RTU 实现柜内温度传感器数据采集、温度上报和超温告警。
6. 基于 MQTT 设计设备接入协议，实现心跳、会员事件、订单事件、柜门状态、温度数据、告警和远程控制。
7. 基于 SQLite 实现会员、商品、订单、余额流水、柜门记录和温度数据的本地持久化，支持断网缓存与补传。
8. 基于 HTTP + SHA256 实现应用级 OTA，支持版本检测、升级包下载、完整性校验、程序替换和失败回滚。
9. 基于 LVGL 实现嵌入式终端交互界面，支持会员登录、商品选购、订单支付和设备状态显示。
10. 基于 Qt 实现后台管理系统，支持会员管理、订单查看、柜门控制、温度监控、MQTT 消息监控和 OTA 升级推送。
```
